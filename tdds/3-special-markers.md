## 特殊点位 {#special-markers}

本地图中，特殊点位是一个混合了多种实现效果的功能。它的作用是为点位实现基于某些场景下的特殊扩展功能。

它存在多种逻辑与表现形式，包括但不限于：

- 某一地区中，所有点位均包含某功能，但每个点位的功能数据可自定义。
- 某一类物品下，所有点位均包含某功能，但每个点位的功能数据可自定义。
- 某一类物品下，所有点位均满足某个规则。

特殊点位有关的字段为：`AreaVo.code`。

特殊点位直接相关的字段为：`ItemVo.specialFlag` 和 `MarkerVo.extra`。

### 地区配置设计 {#area-config-design}

地区配置为外部资源，它由服务器管理员在服务器直接配置，由 `AreaVo.code` 与对应的地区关联。对于特殊点位功能而言，无需了解 [`TDD-8` 服务器配置][tdd-8] 的详细信息。

[tdd-8]: ./8-server-config.md

### `ItemVo.specialFlag` 设计 {#special-flag-design}

物品的 `specialFlag` 字段，为整数类型，其构成为位掩码位或运算的结果。从低到高的不同位表示的功能如下：

| | 比特位 | 位功能说明 | 掩码 |
| :---: | :--- | :--- | :--- |
| 低位 | 低 1 位 | 是否是传送点位。 | `1 << 0` |
| &dArr; | 低 2 位 | 是否是可自定义图标点位。 | `1 << 1` |
| &dArr; | 低 3 位 | 是否是洞口点位。 | `1 << 2` |
| 高位 | 低 4 位 | 是否是对话点位。 | `1 << 3` |

具体功能会在后续章节说明。

### `MarkerVo.extra` 设计

点位的 `extra`，为一个复杂对象。在实际使用中，`MarkerVo.extra` 的顶层字段并不保证全部存在。其构成和功能会在后续章节详细说明。

### Q & A {#special-markers-qa}

- 为什么需要使用地区代码而非地区名称或地区ID确定地区条目？
  - 此问题已在 [`TDD-2` 数据模型 - 地区模型 - Q & A][tdd-2-area_model_qa] 中给出解答。
- 为什么特殊点位的设计，涉及到物品的 `specialFlag` 和点位的 `extra`？
  - 由于特殊点位存在以下两类情况：
    - 对某一类物品下所有点位的特殊设置。
    - 对某一类物品下点位的特殊配置自定义功能。
  - 因此，特殊点位的设计，需要同时具有能分离出符合条件的物品，又需要对点位有自定义配置，因此同时需要 `ItemVo.specialFlag` 和 `MarkerVo.extra`。
- 为什么 `ItemVo.specialFlag` 是个掩码位整数，而 `MarkerVo.extra` 是个复杂对象？
  - `ItemVo.specialFlag` 的功能是用以过滤不同功能点的物品，且可同时存在多种特性，所以使用掩码位整数进行处理。而 `MarkerVo.extra` 的功能是为特殊点位添加功能的自定义配置，因此使用复杂对象。
- 为什么 `ItemVo.specialFlag` 使用 `number` 掩码位整数，而不使用 `string[]`？
  - `specialFlag` 中的每一个功能仅存在“有此特性”和“无此特性”两种可能，且功能点固定，功能间彼此独立。
  - 此字段需要存入数据库，使用掩码位整数，既节约存储空间，又便于数据库内的直接操作。
  - 在当前业务场景下，掩码位整数是兼容性最强且最简单的实现方案。
- 如何判断一个物品中的特殊点位，是否具有某个功能？
  - 根据位运算规则，可使用与对应掩码位与的方式，判断某物品是否具有对应功能。

## 特殊点位功能 {#special-marker-funcs}

下列说明中，非关键字段将会省略。

### 可传送点位 {#smf-item-teleport}

- 判定条件：
  - `ItemVo.specialFlag AND (1 << 0) > 0`
- 实现功能：
  - 对应物品，不显示在物品选择列表中。
  - 对应物品内点位，可进行标记，但可能需要调整对勾的显示位置。
  - 对应物品内点位，标记后不高亮显示/隐藏点位标记。

### 自定义图标点位 {#smf-item-customize-icon}

- 判定条件：
  - `ItemVo.specialFlag AND (1 << 1) > 0`
- 实现功能：
  - 对应物品，不显示在物品选择列表中。
  - 对应物品内点位，仅可进行基础的聚焦点击，不可进行其他交互与任何的数据处理。
  - 对应物品内点位，渲染图标时忽略物品图标，而使用 `MarkerVo.extra` 中的配置，以实现每个点位的图标自定义。
- `MarkerVo.extra` 配置：
  ``` typescript
  interface MarkerExtraVo {
    iconOverride: {
      id: number;       // 图标ID
      minZoom: number;  // 最小可见缩放级别
      maxZoom: number;  // 最大可见缩放级别
    }
  }
  ```
  - `图标ID`：参见 [`TDD-7` 图标数据]。
  - `最小可见缩放级别`、`最大可见缩放级别`：仅当地图缩放比例在此区间之内时，才可显示自定义图标。对于不同的地图实现方案，此级别数据可能需要进行对应的数学转换。
- 自定义图标渲染时，按照 [`TDD-2` 数据模型 - 物品模型][tdd-2-item_model] 中，常规图标渲染模式进行渲染。

[tdd-2-item_model]: ./2-data-models.md#item-model

### 洞口点位 {#smf-item-cave}

- 判定条件：
  - `ItemVo.specialFlag AND (1 << 2) > 0`
- 实现功能：
  - 无明显实现功能，仅标记作用。
  - 部分功能需要依赖此标记实现，会在具体功能中提及。

### 对话点位 {#smf-item-dialog}

- 判定条件：
  - `ItemVo.specialFlag AND (1 << 3) > 0`
- 实现功能：
  - 对应物品内点位，渲染时忽略部分点位字段，使用 `MarkerVo.extra` 中的配置，独立显示
  - 具体的功能实现参见 [`TDD-10` 对话点位][tdd-10]。
- `MarkerVo.extra` 配置：参见 [`TDD-10` 对话点位][tdd-10]。

### 1.6海岛点位 {#smf-marker-16island}

- 判定条件:
  - `AreaVo.code = 'A:APPLE:1_6'`
- `MarkerVo.extra` 配置：
  ``` typescript
  enum MarkerExtra16Stages = {
    STAGE1 = '1',   // 第一阶段
    STAGE2 = '2',   // 第二阶段
  }

  interface MarkerExtraVo {
    '1_6_island': MarkerExtra16Stages[];
  }
  ```
- 实现功能：
  - 当1.6海岛切换至第一阶段时，仅显示包含 `'1'` 的点位。
  - 当1.6海岛切换至第一阶段时，仅显示包含 `'2'` 的点位。

### 2.8海岛点位 {#smf-marker-28island}

- 判定条件:
  - `AreaVo.code = 'A:APPLE:2_8'`
- `MarkerVo.extra` 配置：
  ``` typescript
  interface MarkerExtraVo {
    '2_8_island': {
      island_name: string;      // 岛屿归属
      island_state: string[];   // 岛屿形态
    }
  }
  ```
  - `岛屿归属` 和 `岛屿形态` 的可用值为：
    - `''`: 无岛屿绑定。
    - `'ww'`：危危岛
      - `'0'`：初始
      - `'1'`：沉入水面
    - `'pp'`: 破破岛
      - `'0'`：初始
      - `'2'`：低
      - `'1'`：高
    - `'ss'`：双双岛
      - `'0'`：初始
      - `'3'`：最终形态
    - `'bd'`：布丁岛
      - `'0-0'`：初始
      - `'0-1'`：任意-优悠
      - `'0-2'`：任意-浮光
      - `'0-3'`：任意-磐固
      - `'1-0'`：优悠-任意
      - `'2-0'`：浮光-任意
      - `'3-0'`：磐固-任意
      - `'1-1'`：优悠-优悠
      - `'1-2'`：优悠-浮光
      - `'1-3'`：优悠-磐固
      - `'3-1'`：磐固-优悠
      - `'3-2'`：磐固-浮光
      - `'3-3'`：磐固-磐固
      - `'2-1'`：浮光-优悠
      - `'2-2'`：浮光-浮光
      - `'2-3'`：浮光-磐固
- 实现功能：
  - 当2.8海岛的点位不绑定岛屿时，点位常显。
  - 当2.8海岛的点位绑定岛屿时，当对应岛屿切换到特定形态时，正常显示符合岛屿形态的绑定点位，弱化显示不符合岛屿形态的绑定点位。

### 分层点位 {#smf-marker-layered}

- `MarkerVo.extra` 配置：
  ``` typescript
  interface MarkerExtraVo {
    is_underground: boolean;    // 是否是分层点位
    is_global: boolean;         // 是否是非地面点位
    region_levels: string[];    // 分层区域
  }
  ```
  - `非地面层级`：有一些点位，并非地面点位，但无法归类到分层层级中的任何层级，因此使用【非地面层级】进行标记。参见 [`TDD-9` 分层层级][tdd-9]。
- 实现功能：
  - 当点位为分层点位时，在点位上显示分层角标。
  - 当点位为非地面点位时，若当前地图为提瓦特地面地图，则高亮对应点位的分层角标。
  - 当点位不为非地面点位时，若绑定的分层区域显示在地图上，则高亮对应点位的分层角标。其他细节可根据 [`TDD-9` 分层层级][tdd-9] 中的具体实现进行调整。
